# Taskeroo

## Description
This feature allows users to create tasks, like a lightweight JIRA.
The name of the module is taskeroo.
Since this will be a massive mono repo, scope the taskeroo part in modules (backend) and /taskeroo route (frontend)

## Workflow
Tasks will be discovered by AI Agents. Agents can start working on tasks by:
- assigning them to themselves (must provide an agent name)
- assigning a session id (this is an AI agent concept, where the session id tracks all changes made)

## Functional
- Create task. Needs:
  - task name
  - task description
  - optionally, it can be assigned by providing:
    - assignee
    - session id
- Edit tasks:
  - can edit description
- Assign tasks:
  - by adding asingee
- Delete task
- List tasks
  - all
  - by assignee
  - by session id
- Add comments on tasks
  - needs name of commenter
- Change task status
  - options: not started - in progress - for review - done
  - in progress needs to be assigned to someone
  - done needs a comment
- All of the above can to be done via the API
- All of the above can to be done via a UI

## Technical
- Tasks are persiste in sqlite
- Entities, services, controllers, dto. All defined following best practices in `docs/how-to-guides` and `docs/best-practices`

## UI requirements
- UI must be clean - light/white background with light lines and soft curves
- User can click on tasks and see stuf
- User can create tasks that go to the `not started` column
- Real time connection via websocket or something such that the UI immediately displays changes made by the API
